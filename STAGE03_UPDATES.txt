1) Theme Switcher (Logged-in UI)

2) Theme Consistency Rules
- Public/non-user pages are forced to bright mode only.
- On logout, theme resets automatically to bright mode.

3) UI/Navigation Fixes
- Fixed Sign Up button text contrast issue.
- Fixed About Us navbar inconsistency for logged-in users.
- Added consistent logged-in nav rendering in protected views.

4) Session Security Improvement
- Implemented single active session per user (one device/browser at a time).
- If same account logs in elsewhere, user gets confirmation prompt:
  - Yes: old session invalidated, new session activated.
  - No: login blocked, old session remains active.

5) Admin Portal (New Role Experience)
- Added admin-only dashboard route: /admin/dashboard.html
- Added role guard so only admin users can access admin portal.
- Added Admin Portal nav link visible only to admin users.
- Made admin navbar placement consistent across pages.

  - error count

  - forwarded items
  - report downloads

7) Admin/User Dashboard Separation
- Admin users are redirected from /dashboard.html to /admin/dashboard.html
